# PRD-001：LLM 配置可用性检测

## 元信息

- 状态：已完成
- 作者：Ongrid
- 日期：2026-07-23
- 关联 issue：用户直接需求
- 关联 Epic：不适用
- 关联 ADR：不适用（复用现有 LLM 请求链路，不引入架构决策）
- 关联 HLD：不适用（不新增服务或跨域模块）

## 背景

管理员当前在“设置 → LLM 模型”中只能保存 API Key、Base URL 和模型名。配置错误要等到聊天或自动诊断真正调用模型时才暴露，而且页面只显示上游原始错误，无法快速判断是 Key、模型、网络、TLS、额度还是限流问题。

目标用户是负责部署和维护 Ongrid 的管理员。

## 目标

- 对非空 LLM 配置中的每个模型执行真实的最小 Chat Completions 请求，任一失败均不落库。
- 对已知错误场景返回稳定、可机器识别的错误码，并以单元测试覆盖主要分类和保存门禁。
- 探测最长 20 秒，API Key 不进入日志、响应详情或持久化探测记录。
- 校验和保存由同一个服务端请求完成，四个配置字段在一个数据库事务中原子写入。

## 功能需求

### 核心功能

- 每张 LLM 提供商卡片提供“测试配置”操作，直接检测当前未保存草稿。
- 点击“保存”时由 Manager 校验同一份草稿中的完整模型列表；只有全部通过后才在一个事务中保存 API Key、Base URL、默认模型和模型列表。
- API Key 为空表示禁用该提供商，允许直接保存空值，不发起探测。
- 探测使用与正式聊天一致的 OpenAI-compatible Chat Completions 路径、Base URL 规范化和智谱 JWT 鉴权。
- 页面区分并解释以下原因：缺少字段、URL/协议错误、鉴权失败、权限不足、模型不存在、额度不足、限流、DNS、连接、TLS、超时、端点不存在、上游不可用、响应无效和其他上游错误。
- 修改任一配置字段后，旧的探测成功状态立即失效。
- 手动测试成功不能绕过保存接口的服务端校验，避免测试值和最终落库值不一致。

### 边界情况

- 命名提供商 Base URL 留空时使用服务端已有默认端点；`custom` 必须填写 Base URL。
- Base URL 只允许 `http` 或 `https`，必须包含 host，禁止 userinfo、query 和 fragment。
- 只允许管理员调用探测 API；未知字段和超大请求体直接拒绝。
- 自定义提供商延续现有能力，允许管理员指向受信任的内网模型服务；页面明确该请求由 Manager 发起。
- 上游返回内容仅保留截断后的安全摘要，并对当前 API Key 做二次脱敏。
- 模型列表去空、去重且最多 32 个；默认模型必须包含在列表中。

## 非功能需求

- **性能 / 可用性**：整组模型校验共享 20 秒超时；不重试，避免额外费用和放大限流。
- **安全**：API Key 仅存在于本次 HTTPS 请求内存中，不落库、不审计、不记录；服务端校验 URL 结构。探测接口为管理员专用。
- **运维**：无数据库迁移、无新增依赖。回滚时移除探测路由和前端保存门禁即可，既有配置数据不受影响。
- **兼容性**：未写数据库覆盖时继续继承环境变量；已写入的空 Key 是显式禁用覆盖，动态目录为空时不得退回启动时客户端。

## API 变更

新增 `api/manager/setting/v1/setting.proto` 中的两个契约：

```text
POST /api/v1/integrations/llm/test
POST /api/v1/integrations/llm/validate-and-save
```

请求均包含 `provider`、`api_key`、`base_url`、`default_model`、`models`。校验完成统一返回 HTTP 200 与 `valid/code/detail/latency_ms`；保存接口另返回 `saved/disabled`。认证、请求格式或服务未接线仍使用对应 4xx/5xx。

## 数据库变更

无 schema 变更。探测结果不持久化；已有 `system_settings` 四行配置通过单个事务批量 upsert。

## 依赖与阻塞

- **依赖**：现有 `internal/pkg/llm` OpenAI-compatible 客户端及设置页面。
- **阻塞**：无。

## 风险与假设

- **已知风险**：真实探测会产生极少量 token 消耗；通过最小提示词且不重试控制成本。
- **已知风险**：管理员可配置内网 URL，存在服务端出站访问能力；这是现有自定义模型功能的必要能力，继续限定为管理员并执行协议与 URL 结构白名单校验。
- **假设**：支持的厂商均继续提供 OpenAI-compatible Chat Completions 接口。
- **未知项 / TODO**：暂不持久化最近一次验证时间，避免把一次成功误当成长期健康状态。

## 验收标准

- [x] 正确 Key、Base URL 和完整模型列表返回“配置可用”及总耗时。
- [x] 错误 Key、错误模型、额度/限流、DNS/连接、TLS、超时及错误端点分别显示不同原因。
- [x] 保存非空配置时服务端逐一探测全部模型，失败或事务中途报错时任何字段均不写入。
- [x] 空 Key 能覆盖环境变量配置并让运行时目录真正移除该提供商。
- [x] API Key 不出现在探测响应、日志或测试快照中。
- [x] API 与错误分类单元测试带 `-race` 通过。
